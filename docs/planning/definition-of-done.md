# Definition of Done

For any story to be considered “done”, the following must be true:

1. **Accepted by the Product Owner / Facilitator** – Acceptance criteria are met and demonstrated in a working environment.
2. **Code Review Completed** – At least one team member has reviewed the pull request and approved it.
3. **CI Pipeline Passes** – All automated checks (build, lint, tests) pass without errors.
4. **Tests Added or Updated** – Appropriate unit or smoke tests exist for the new functionality.
5. **Documentation Updated** – Any relevant docs (README, runbook, architecture diagrams) reflect the change.
6. **No Critical Bugs** – There are no open critical defects associated with this story.
