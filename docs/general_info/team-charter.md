## 3. Roles and Responsibilities

We assign roles to ensure responsibility is clear.  
All members are expected to contribute to coding and core engineering work.

### Roles

**Coordinator (Facilitator)**
- Organizes meetings (planning, demo, retro)
- Tracks progress and blockers
- Keeps team aligned with timeline

**Engineering Lead**
- Oversees system structure and integration
- Ensures the system runs end-to-end
- Supports technical decisions

**Quality & Process Owner**
- Ensures Definition of Done (DoD) is followed
- Checks PR review, CI, and basic tests
- Maintains workflow discipline

**Documentation & Handover Owner (Rotating)**
- Maintains README, runbook, and docs
- Records key decisions (ADRs if applicable)
- Prepares demo narrative and updates

**Jack or Jill (of All Trades)**
- Helps team in all areas
- Comes up with ideas

## 5. Workflow Agreement

We agree to follow:

- All work tracked as issues
- All changes via Pull Requests
- At least **1 reviewer per PR**
- CI must pass before merge
- Work must meet Definition of Done (DoD)

## 6. Contribution Expectations

Each member is expected to:

- Contribute regularly every week
- Communicate blockers early
- Participate in planning, demo, and retro
- Be able to explain their work
